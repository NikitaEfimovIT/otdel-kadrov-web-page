# План реализации сайта гильдии «Отдел кадров»

> Как из hi-fi макетов Claude Design (сайт + админка) собрать рабочий продукт.
> Релиз — «всё сразу»: публичный сайт + управление контентом + бэкенд.

---

## TL;DR

- **Фронтенд:** React SPA (Vite + TypeScript), хостинг — **GitHub Pages** (бесплатно, из РФ обычно открывается без VPN).
- **Бэкенд/CMS:** **Strapi 5** (headless CMS — админка из коробки + REST/GraphQL API).
- **БД + хостинг API:** **российский managed** — Timeweb Cloud (Managed PostgreSQL + App Platform) или Yandex Cloud. **Не Supabase/Neon/Vercel/Cloudflare** — их сейчас режут в РФ.
- **Домен:** свой (лучше `.ru`), DNS на RU-доступном провайдере (не Cloudflare).
- **Главное ограничение проекта:** всё, что дёргает браузер пользователя из РФ, должно открываться **без VPN**. Это диктует выбор хостинга (см. §4).

---

## 1. Что реализуем

Три части, один релиз:

1. **Публичный сайт** — воссоздать 8 секций макета пиксель-в-пиксель на React, на живых данных из API (hero, о гильдии, прогресс рейда, новости, опросы, ссылки, набор, футер).
2. **Управление контентом** — чтобы офицеры добавляли новости/опросы/ссылки и видели заявки без правки кода.
3. **Бэкенд** — API + БД + авторизация, которые всё это питают.

Макеты — это дизайн-референс (`.dc.html` + `support.js`), а не прод-код. Логику из класса `Component` переносим в React-компоненты, статические массивы (`news`, `polls`, `links`, `apps`) заменяем запросами к API. `support.js` в прод не идёт.

## 2. Итоговый стек

| Слой | Технология | Почему |
|---|---|---|
| Фронтенд | React 18 + Vite + TypeScript | SPA, как ты и хотел; быстрый билд; статика под GitHub Pages |
| Стили | CSS-переменные из README (токены) + CSS Modules / Tailwind | Токены уже заданы дизайном — переносим 1:1 |
| Иконки | `lucide-react` | Ровно та же библиотека, что в макете |
| Шрифты | Manrope + JetBrains Mono (Google Fonts, с самохостом как фолбэк) | Как в дизайне; самохост — на случай троттлинга Google в РФ |
| CMS/бэкенд | Strapi 5 (Node.js) | Готовая админка + REST/GraphQL; content-types под нашу модель данных |
| БД | PostgreSQL (managed) | Стандарт; managed у RU-провайдера |
| Хостинг фронта | GitHub Pages | Бесплатно, из РФ открывается |
| Хостинг API+БД | Timeweb Cloud / Yandex Cloud | Managed + доступ из РФ без VPN + 152-ФЗ |
| Авторизация офицеров | Strapi admin (email/пароль) | Не зависит от Discord (который в РФ душат) |

> Про «CMS типа strip» — ты имел в виду **Strapi**. Альтернативы того же класса: **Directus**, **Payload CMS**, **KeystoneJS**. Рекомендую Strapi (самое зрелое, огромное комьюнити, RU-гайды). Если захочешь — сравню их отдельно.

## 3. Архитектура

```
Браузер игрока (РФ, без VPN)
        │
        ├─ 1. Грузит статику ──────────►  GitHub Pages  (React SPA)
        │                                   github.io / свой домен
        │
        └─ 2. Запросы к данным ────────►  api.наш-домен.ru
                                            └─ Strapi 5  (Timeweb/Yandex, РФ)
                                                 └─ PostgreSQL (managed, РФ)

Офицер → та же статика → /admin (Strapi) → правит контент
Заявки/новые заявки → вебхук в Discord гильдии (уведомление)
Прогресс рейда → v1 вручную в Strapi → v2 авто из Raidsmith
```

Ключ: **и статика, и API должны быть RU-доступны**. Если API уедет на заблокированный хостинг — статика откроется, но сайт будет «пустым» (запросы виснут). Именно поэтому API на российском managed.

## 4. Критично: доступ из РФ без VPN

Проверил актуальную (июль 2026) ситуацию. Коротко — чего **избегать** и что **брать**:

| Сервис | Статус в РФ (2026) | Вердикт |
|---|---|---|
| **Vercel** (кастомные домены) | Блокируется — общие IP попадают под блок; `*.vercel.app` работает, кастомный домен часто нет | ❌ не для API/сайта |
| **Supabase / Neon** | Запросы виснут без VPN, админка не грузится | ❌ не как БД/бэкенд |
| **Cloudflare** (proxy/Pages) | Троттлинг и DPI-деградация, лимит отдачи ~16 КБ с 9 июня; RKN советует уходить на RU-хостинг | ❌ не как CDN/прокси |
| **GitHub / GitHub Pages** | RKN отрицает блокировку; в целом доступен, но в мае 2026 были эпизоды троттлинга (10–16% обрывов) | ✅ ок для статики, с фолбэком |
| **Discord** | Периодически душат в РФ | ⚠️ не делать критичным для входа |
| **Timeweb Cloud / Yandex Cloud** | Российские, Tier III, 152-ФЗ, доступ гарантирован | ✅ для API + БД |

Правила, которые из этого следуют:
- API и БД — только на российском managed (Timeweb/Yandex).
- Не завязывать вход офицеров на Discord (берём Strapi email/пароль; Discord-вход — опционально).
- Google Fonts продублировать самохостом (шрифты в билд), чтобы верстка не поехала при троттлинге.
- **Фолбэк по фронту:** если GitHub Pages начнёт троттлить — переносим статику на тот же RU-хостинг (Timeweb статик/S3 + CDN). Архитектура это позволяет без переписывания.
- **152-ФЗ:** заявки содержат перс. данные (ник/BattleTag, коммент) → хранить в РФ. Timeweb/Yandex это закрывают.

## 5. Хостинг и инфраструктура

**Рекомендую Timeweb Cloud** (проще и дешевле для нашего масштаба):
- Managed PostgreSQL (версии 14–18, бэкапы, SLA 99.98%).
- App Platform для Node.js: авто-CI/CD из GitHub, лог/метрики, откаты в один клик — туда деплоим Strapi.
- 152-ФЗ, Tier III ЦОД.

**Yandex Cloud** — если нужен запас по надёжности/масштабу: Managed PostgreSQL + Serverless Containers (Strapi в контейнере, первый 1 млн вызовов/мес бесплатно) + Object Storage под медиа (есть комьюнити-провайдер загрузки для Strapi).

**Домен и DNS:**
- Зарегистрировать домен (лучше `.ru`, напр. через reg.ru) — RU-домен стабильнее по доступу.
- DNS держать на RU-доступном NS (reg.ru / Yandex), **не на Cloudflare**.
- `наш-домен.ru` → GitHub Pages (или RU-статик); `api.наш-домен.ru` → Strapi на Timeweb.
- GitHub Pages поддерживает кастомный домен + авто-HTTPS.

## 6. Бэкенд: Strapi (content-types)

Модель данных из README ложится на коллекции Strapi почти 1:1:

| Strapi type | Поля | Заметки |
|---|---|---|
| `news` (collection) | title, body (rich), tag (enum: Рейд/Набор/Ивент/Объявление), date, status (draft/published), views | Черновики = draft & publish Strapi |
| `poll` (collection) | question, status (active/closed), endsAt, multiple (bool), anonymous (bool), → options | |
| `poll-option` (component/relation) | text, votes (int) | Вложенный компонент или связь |
| `vote` (collection) | poll, option, voterToken/user | Уникальность голоса — см. §7 |
| `application` (collection) | battletag, wowClass, spec, logsUrl, comment, status (new/reviewing/accepted/rejected), createdAt | Публичный `create`, приватный список |
| `role` (collection) | key, name, open (bool) | Тоггл «нужен/закрыт» |
| `link` (collection) | name, icon, url, visible (bool), order | Drag-порядок = поле order |
| `guild-setting` (single type) | name, region, faction | Одна запись |
| `raid-progress` (single type) | raidName, difficulties[], bosses[], mplusBestKey, mplusRating | v1 вручную, v2 из Raidsmith |

Права (Users & Permissions):
- **Public role:** read — news(published), polls, links(visible), roles, guild-setting, raid-progress; create — application, vote. Всё остальное закрыто.
- **Officer:** через админку Strapi (полный CRUD).

Плюс **вебхук**: на `application.create` — POST в Discord-канал гильдии («новая заявка: BattleTag, класс»). Дёшево и очень удобно офицерам.

## 7. Авторизация

Три разных случая — не путать:

1. **Офицеры (управление контентом).** Вход в админку Strapi по **email/паролю**. Надёжно и не зависит от Discord. (Discord-вход по макету можно добавить как второй способ через провайдер Strapi, но не делать единственным — Discord в РФ нестабилен.)
2. **Публичные заявки.** Аноним, просто `POST /applications`. Валидация BattleTag (`Имя#0000`), обязательные поля, антиспам (honeypot + rate-limit).
3. **Голосование в опросах.** Тут развилка «один голос на человека»:
   - **v1 (как в макете, низкий трение):** токен в localStorage + rate-limit по IP. Не защищает от накрутки, но честно для дружеского опроса.
   - **v2 (настоящая целостность):** голосование только для залогиненных (Discord или гильдийский пароль), `unique(poll, user)`. Наружу — только агрегат (анонимно для других).
   Рекомендую стартовать с v1, заложив поле `voterToken`, чтобы поднять до v2 без миграции.

## 8. Фронтенд (React SPA)

- **Каркас:** Vite + React + TS. Токены из README → `:root` CSS-переменные (`--bg`, `--accent` и т.д.). Шрифты самохостом.
- **Роутинг на GitHub Pages:** SPA + Pages требует либо **HashRouter**, либо трюк с `404.html` (копия `index.html`) для BrowserRouter. Публичный сайт — одностраничник с якорями (`#hero` … `#recruit`), так что роутинг минимальный; scroll-spy через `IntersectionObserver` (как в макете).
- **Компоненты:** каждая секция макета → компонент; данные тянем из Strapi (`fetch`/react-query). Заглушечные массивы заменяем ответами API.
- **Состояния из макета:** голосование (`voted/choice`), сабмит формы (`submitted`), статус набора (`recruitingOpen` из `guild-setting`/`role`), тумблер Raidsmith (`showRaidsmith`).
- **Мобильная адаптация:** макетов мобилки в пакете НЕТ (десктоп-first). Проектируем сами по брейкпоинтам 640/768/1024/1280: сетки 3→2→1, hero-бенто 4→2→1, nav → бургер, таблицы → карточки. Заложить время.
- **Админка:** в v1 — встроенная админка Strapi (быстро, из коробки). Кастомный «Кабинет офицера» по макету (сайдбар, дашборд, редакторы) — **опциональная фаза 2**: отдельный SPA-раздел против того же API. Это главное «или-или» по объёму (см. §13).

## 9. Прогресс рейда и задел под Raidsmith

- **v1:** `raid-progress` (single type) заполняется вручную в Strapi — боссы, сложности, M+ рейтинг. Секция «Прогресс» и бенто-плитки hero читают его.
- **v2 (Raidsmith):** заменяем ручной ввод авто-подтягиванием из нашего сервиса (прогресс, ростер/штат, посещаемость, логи). В макете уже заложены места-заглушки «Виджет Raidsmith · СКОРО» и кнопка «Войти через Raidsmith» — включаются флагом `showRaidsmith`. По сути нужен эндпоинт-мост Raidsmith→сайт (или общий API, если делаем на том же стеке).

## 10. Интеграции

- **Discord:** вебхук на новые заявки (уведомление офицерам) — берём сразу, полезно и просто. OAuth-вход — опционально/позже.
- **raider.io / Warcraft Logs:** в v1 — просто ссылки (плитки в секции «Ссылки»). Позже — подтягивать прогресс/рейтинг по их API (или через Raidsmith).
- **Raidsmith:** см. §9.

## 11. План работ по этапам

Даже при «всё сразу» держим внутренний порядок (можно частично параллелить). Оценки — грубые, в рабочих днях, для одного разработчика.

1. **Инфраструктура (1–2 дн):** домен + DNS, RU-хостинг, Managed Postgres, деплой пустого Strapi, HTTPS, CORS фронт↔API.
2. **Бэкенд-модель (2–3 дн):** content-types (§6), права Public/Officer, сиды (текущие новости/ссылки/прогресс), вебхук в Discord.
3. **Каркас фронта (1–2 дн):** Vite+React+TS, токены/шрифты/lucide, деплой-пайплайн на Pages, роутинг/scroll-spy.
4. **Публичный сайт (4–6 дн):** 8 секций пиксель-в-пиксель на живых данных.
5. **Опросы + форма заявки (2–3 дн):** голосование (v1), сабмит с валидацией и состоянием успеха.
6. **Управление контентом (1 дн v1 / +5–8 дн фаза 2):** v1 — настроить админку Strapi; фаза 2 — кастомный «Кабинет офицера».
7. **Мобильная адаптация (2–4 дн):** брейкпоинты, бургер, карточные таблицы.
8. **QA и запуск (2–3 дн):** кросс-браузер, **проверка доступа из РФ без VPN**, Lighthouse/SEO для визитки, метатеги/OG, sitemap.

**Итого v1 (админка Strapi):** ≈ 15–24 дн. **С кастомной админкой (фаза 2):** +5–8 дн.

## 12. Риски и как снимаем

- **Троттлинг GitHub Pages в РФ** → фолбэк: перенос статики на RU-хостинг (архитектура готова); свой `.ru` домен.
- **Google Fonts душат** → самохост шрифтов в билде.
- **Discord нестабилен в РФ** → вход офицеров по email/паролю, Discord не критичен.
- **152-ФЗ (перс. данные заявок)** → хранение в РФ (Timeweb/Yandex).
- **Накрутка опросов** → v1 честный best-effort; поднять до авторизованного голосования при необходимости.
- **Нет мобильных макетов** → заложено время в §11; при желании — запросить мобильные экраны у Claude Design.

## 13. Решения, которые нужно принять

1. **Админка:** v1 на встроенной админке Strapi (быстро) ИЛИ сразу кастомный «Кабинет офицера» по макету (+5–8 дн)? — влияет на срок сильнее всего.
2. **Хостинг API:** Timeweb Cloud (рекомендую) или Yandex Cloud?
3. **Домен:** какое имя берём (и `.ru`?).
4. **Голосование:** v1 best-effort или сразу авторизованное?
5. **Discord:** дать ссылку-приглашение/ID сервера и канал под вебхук заявок?
6. **Акцентный цвет:** оставляем «арканский фиолет» из макета или возвращаем золото-на-синем под герб?
7. **Финальные ссылки** (raider.io, WCL, WoWProgress, Twitch, YouTube) для секции «Ссылки».
